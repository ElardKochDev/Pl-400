# PROMPTS de BOSSES mejorados — PL-400 (tandas de 15)

> **CAMBIO CLAVE vs la tanda anterior:** los bosses deben salir como los ENEMIGOS (que quedaron perfectos): **una figura por celda sobre FONDO PLANO uniforme MUY OSCURO** (gris-azulado casi negro). **SIN trono, SIN arco, SIN pedestal, SIN sala ni suelo, SIN marco, y SOBRE TODO SIN TEXTO / rótulos / placas / etiquetas.** Solo el personaje, de cuerpo entero, centrado, con márgenes claros y sin tocar los bordes de su celda. Así los puedo recortar con transparencia igual que los enemigos.

**Estilo:** pixel-art JRPG 16-bit (SNES), contornos limpios, colores vivos. Los bosses son **GRANDES, imponentes y detallados** (más que los enemigos), con pose amenazante y un aura de energía a su alrededor (la energía NO cuenta como fondo: el fondo sigue siendo plano). Temática: desarrollador de Microsoft Power Platform. Un pequeño accesorio/prop temático en la mano está bien (p. ej. un tomo, un orbe, un báculo), **pero sin texto legible en él**.

Entrégame cada tanda como UNA imagen. Yo la troceo y la nombro con las claves indicadas.

## TANDA 1 — Bosses 1–15  →  archivo `bosses_batch1.png`

Cuadrícula **3 columnas × 5 filas** (15 bosses), fondo plano oscuro uniforme, una figura por celda:

1) **The Imperative Warden** (Mundo 1 · Hall of Variables · paleta azul acero) — Imperative logic defines HOW step by step (Set, Navigate); declarative defines WHAT.
2) **Master of Patch** (Mundo 1 · Forge of Patch · paleta azul acero) — Patch(DataSource, BaseRecord, ChangeRecord) writes precise changes to data.
3) **Choicemaster** (Mundo 1 · Gallery of Choices · paleta azul acero) — Use global choices for reuse and lookups for real relationships between tables.
4) **Warden of Relations** (Mundo 1 · Keep of Relations · paleta azul acero) — Dataverse relationships (1:N, N:N) model related data; Power Apps navigates them.
5) **Formula Sage** (Mundo 1 · Sanctum of Expressions · paleta azul acero) — Power Fx is the declarative, Excel-like typed expression language of Power Apps.
6) **Delegation Overlord** (Mundo 2 · Bastion of Delegation · paleta verde-cian) — Prefer delegable functions and data sources so the server filters and sorts data.
7) **The All-Seeing Monitor** (Mundo 2 · Observatory of Monitor · paleta verde-cian) — Use Monitor and Application Insights to find and fix performance and errors.
8) **Herald of the Platform** (Mundo 2 · Nexus of the Platform · paleta verde-cian) — Power Platform = Power Apps + Automate + BI + Pages + Copilot Studio over Dataverse.
9) **Toolsmith of ALM** (Mundo 2 · Workshop of Tools · paleta verde-cian) — Package with SolutionPackager/CLI; managed to deploy, unmanaged to develop.
10) **Extensibility Archmage** (Mundo 2 · Forge of Extensibility · paleta verde-cian) — Extend Dataverse via Web API, Organization Service, plug-ins and Custom APIs.
11) **Scriptbinder** (Mundo 3 · Atrium of Client Script · paleta violeta) — Client scripting (JavaScript) reacts to form events using the Client API context.
12) **Xrm Overlord** (Mundo 3 · Spire of the Xrm Object · paleta violeta) — The Xrm object exposes App, Navigation, Utility and WebApi for global operations.
13) **Component Architect** (Mundo 3 · Foundry of Components · paleta violeta) — The Power Apps component framework builds reusable code components for apps.
14) **The Codewright** (Mundo 3 · Laboratory of Code Components · paleta violeta) — Implement updateView/getOutputs, package into a solution and deploy the component.
15) **Keeper of Dataverse** (Mundo 4 · Crypt of Dataverse · paleta ámbar/rojo) — Extend Dataverse with .NET plug-ins registered on pipeline events and messages.

_Claves (celda por celda, izq→der / arriba→abajo):_ `b_p1m1`=The Imperative Warden, `b_p1m2`=Master of Patch, `b_p1m3`=Choicemaster, `b_p1m4`=Warden of Relations, `b_p1m5`=Formula Sage, `b_p2m1`=Delegation Overlord, `b_p2m2`=The All-Seeing Monitor, `b_p2m3`=Herald of the Platform, `b_p2m4`=Toolsmith of ALM, `b_p2m5`=Extensibility Archmage, `b_p3m1`=Scriptbinder, `b_p3m2`=Xrm Overlord, `b_p3m3`=Component Architect, `b_p3m4`=The Codewright, `b_p4m1`=Keeper of Dataverse

## TANDA 2 — Bosses 16–27  →  archivo `bosses_batch2.png`

Cuadrícula **3 columnas × 4 filas** (12 bosses), fondo plano oscuro uniforme, una figura por celda:

1) **Plug-in Overlord** (Mundo 4 · Vault of Plug-ins · paleta ámbar/rojo) — A plug-in implements IPlugin.Execute and reads IPluginExecutionContext.
2) **Warden of the Web API** (Mundo 4 · Gateway of the Web API · paleta ámbar/rojo) — Call Dataverse over HTTP with OData and OAuth bearer tokens from Entra ID.
3) **Integration Overlord** (Mundo 5 · Nexus of Azure · paleta azul-cian eléctrico) — Publish Dataverse data to Azure via Service Bus, Functions and messaging.
4) **Reactor Core** (Mundo 5 · Reactor of Functions · paleta azul-cian eléctrico) — Azure Functions run event-driven serverless code that scales on demand.
5) **Binding Artificer** (Mundo 5 · Assembly of Bindings · paleta azul-cian eléctrico) — Define triggers and bindings to wire functions to Azure services.
6) **Portal Warden** (Mundo 5 · Portal of Connectors · paleta azul-cian eléctrico) — Build a custom connector in a solution via the maker portal and an OpenAPI definition.
7) **Connector Smith** (Mundo 5 · Workshop of the Connector · paleta azul-cian eléctrico) — Create custom connectors from VS, APIM or an OpenAPI (Swagger) definition.
8) **Gatekeeper of Auth** (Mundo 6 · Citadel of Authentication · paleta púrpura-dorado) — Secure connectors with OAuth 2.0/Entra ID over an API Management gateway.
9) **Policy Magistrate** (Mundo 6 · Hall of Policies · paleta púrpura-dorado) — Connector policies reshape traffic: set values, convert data, route requests.
10) **Swagger Sage** (Mundo 6 · Archive of OpenAPI · paleta púrpura-dorado) — OpenAPI extensions add dynamic values/schema, test connection and chunking.
11) **Solution Steward** (Mundo 6 · Foundry of Solutions · paleta púrpura-dorado) — Manage solutions with connection references, env variables and ALM pipelines.
12) **Grand Architect of ALM** (Mundo 6 · Citadel of ALM · paleta púrpura-dorado) — Architect solutions with layering, version control and CLI unpack for healthy ALM.

_Claves (celda por celda, izq→der / arriba→abajo):_ `b_p4m2`=Plug-in Overlord, `b_p4m3`=Warden of the Web API, `b_p5m1`=Integration Overlord, `b_p5m2`=Reactor Core, `b_p5m3`=Binding Artificer, `b_p5m4`=Portal Warden, `b_p5m5`=Connector Smith, `b_p6m1`=Gatekeeper of Auth, `b_p6m2`=Policy Magistrate, `b_p6m3`=Swagger Sage, `b_p6m4`=Solution Steward, `b_p6m5`=Grand Architect of ALM
